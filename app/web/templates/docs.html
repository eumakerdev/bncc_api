Autenticação por API key

A API de dados é autenticada por API key enviada no cabeçalho Authorization como Bearer. Gere uma key gratuita no portal self-service após verificar seu e-mail. 

curl -H "Authorization: Bearer SEU_TOKEN_AQUI" \
  https://sua-instancia/api/v1/habilidades/EF05MA07

Requisições sem uma API key válida recebem 401.

Exemplos rápidos

Consultar uma habilidade por código

curl -H "Authorization: Bearer SEU_TOKEN_AQUI" \
  https://sua-instancia/api/v1/habilidades/EM13MAT101

Listar com filtros e paginação

curl -H "Authorization: Bearer SEU_TOKEN_AQUI" \
  "https://sua-instancia/api/v1/habilidades?etapa=ensino_medio&size=20"

Busca semântica com IA (não-oficial)

curl -X POST https://sua-instancia/api/v1/busca-semantica \
  -H "Authorization: Bearer SEU_TOKEN_AQUI" \
  -H "Content-Type: application/json" \
  -d '{"query": "quais habilidades de matemática do 5º ano tratam de frações?"}'

As respostas de busca semântica trazem fontes rastreáveis e são sempre marcadas como conteúdo gerado, não-oficial — os dados determinísticos permanecem a fonte da verdade.

Limites de uso (rate limiting)

CotaLimiteAplica-se a
Determinística60 requisições/minHabilidades, competências, taxonomia, sistema
IA (busca semântica)20 requisições/min e teto de 500/dia/api/v1/busca-semantica

Acima do limite, a API responde 429 com o cabeçalho Retry-After indicando quando tentar novamente. As duas cotas são independentes: exceder a cota de IA não afeta os endpoints determinísticos.

Versionamento

Todos os endpoints ficam sob /api/v1. Mudanças incompatíveis (breaking changes) são publicadas em uma nova versão (/api/v2), nunca dentro da versão atual.
